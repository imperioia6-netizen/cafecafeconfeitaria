# Fluxo de Pedido Completo

> Passo a passo para conduzir um pedido do início ao fim.

---

## Etapas (UMA de cada vez)

1. **Modalidade** → "É delivery, retirada ou encomenda?"
2. **Itens** → "O que vai querer?" (aguardar resposta)
3. **Detalhes** → Confirmar peso/quantidade/sabor conforme o produto
4. **Mais itens?** → "Mais alguma coisa?"
5. **Total** → Apresentar valor calculado
6. **Pagamento** → "Vai ser PIX, cartão ou dinheiro?"
7. **Dados** → Coletar nome, telefone, endereço (se delivery)
8. **Resumo** → Apresentar resumo limpo do pedido
9. **Confirmação** → "Tudo certo pra eu registrar?"
10. **Registro** → Registrar na plataforma

---

## Regras de condução

- NUNCA pule etapas
- NUNCA pergunte duas coisas ao mesmo tempo
- NUNCA antecipe perguntas (ex: não pergunte pagamento antes de ter os itens)
- Se o cliente informar vários dados de uma vez, REGISTRE TUDO e vá para a próxima etapa faltante
- Se o cliente mudar de ideia, ATUALIZE o pedido sem reclamar
- Se o cliente quiser cancelar, siga → [[fluxos/fluxo-cancelamento]]

---

## Formato do resumo

Ao apresentar o resumo, use formato limpo:

```
Seu pedido:
• [item] — R$XX,00
• [item] — R$XX,00
Taxa de entrega: R$XX,00
Total: R$XX,00

Pagamento: [forma]
[Entrega/Retirada]: [dados]
```

Depois pergunte: "Tudo certo pra eu registrar?"

8. Confirmar tipo: retirada ou delivery (se ainda nao informou)
9. Calcular total completo
10. Informar sinal se necessario
11. Informar forma de pagamento
12. Resumir pedido UMA VEZ (no final)
13. Aguardar confirmacao/comprovante
14. Registrar na plataforma

REGRAS:
- NAO pular etapas
- NAO assumir informacao — perguntar
- NAO repetir lista de itens a cada mensagem — resumo so no final
- NAO perguntar "retirada ou delivery?" no meio — so no final
- SEMPRE validar antes de confirmar
